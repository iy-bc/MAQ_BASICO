La creación semántica

El código web semántico se refiere a la práctica de utilizar etiquetas HTML que describen con precisión el significado y el propósito del contenido de una página web. Esto implica utilizar etiquetas HTML que tienen un significado semántico, es decir, que indican el significado de la información que contienen. En lugar de utilizar etiquetas genéricas como <div> o <span>, se utilizan etiquetas específicas como <header>, <nav>, <section>, <footer> y otras etiquetas que se ajustan mejor a la naturaleza del contenido que se está presentando.

Etiquetas de bloques

Las etiquetas HTML de bloques como <div>, <aside>, <nav>, <header>, <section> y <footer> se utilizan para dividir y organizar el contenido de una página web en secciones más pequeñas y significativas. A continuación, se explica el propósito de cada una de estas etiquetas:

  • <div>: La etiqueta <div> se utiliza como un contenedor genérico para agrupar elementos HTML y aplicar estilos CSS a ellos. Es una etiqueta genérica que no tiene ningún significado semántico en sí misma, por lo que se utiliza principalmente como un elemento de diseño.

  • <aside>: La etiqueta <aside> se utiliza para incluir contenido adicional que complementa el contenido principal de la página, pero que no es esencial para su comprensión. Por ejemplo, un aside podría contener anuncios publicitarios o información relacionada que no es el foco principal del contenido de la página.

  • <nav>: La etiqueta <nav> se utiliza para representar una sección de navegación, como un menú de navegación, una lista de enlaces a páginas internas de la web o una tabla de contenido.

  • <header>: La etiqueta <header> se utiliza para representar la sección de encabezado de la página, que suele contener el título, el logotipo, el menú de navegación y otros elementos importantes de la página.

  • <section>: La etiqueta <section> se utiliza para agrupar contenido relacionado y separarlo del resto del contenido de la página. Las secciones pueden ser utilizadas para organizar el contenido en diferentes partes de la página, como una sección para las noticias, otra para los productos y otra para los testimonios de los clientes.

  • <footer>: La etiqueta <footer> se utiliza para representar la sección de pie de página de la página, que suele contener información de contacto, derechos de autor, enlaces a redes sociales y otros elementos de la página que no están relacionados con el contenido principal.

En este ejemplo, se utilizan las siguientes etiquetas de bloques:

  • <header> para el encabezado de la página, que incluye el título y el menú de navegación.
  • <section> para agrupar el contenido principal de la página, que incluye un título, algunos párrafos de texto y una imagen.
  • <aside> para el contenido complementario o tangencial de la página, que incluye una lista de enlaces relacionados.
  • <footer> para el pie de página de la página, que incluye un mensaje de derechos de autor.

Estas etiquetas ayudan a estructurar el contenido de la página de una manera más significativa y semántica, lo que puede ser beneficioso para la accesibilidad, la SEO y la mantención del código.
